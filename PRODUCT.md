# Product

## Register

brand

## Users

Technical and creative operators watching a keynote-style presentation about agent skills as reusable behavior units.

## Product Purpose

Explain agent skills clearly enough that the audience understands the concept, remembers the strongest examples, and sees skill libraries as a practical competitive advantage.

## Brand Personality

Technical, sharp, game-like, opinionated.

## Anti-references

Avoid dense consulting slides, tiny explanatory captions, decorative pull quotes, generic SaaS polish, and text-heavy HUD noise that competes with the main argument.

## Design Principles

- One readable claim per slide.
- Preserve the dark neon game UI, but make the hierarchy calmer.
- Use body text for explanation, not subheadings, captions, or tiny quotes.
- Keep examples concrete and memorable.
- Make broken or missing media obvious to replace.

## Accessibility & Inclusion

Prioritize readable type size, high contrast, and clear hierarchy on a fixed 1280 by 720 presentation canvas.
