◆ AGENT SKILLS// DEFINITION02 / 12
What is an Agent Skill?
The fundamental unit of reusable agent behavior
A SKILL = TRIGGER + INSTRUCTIONS + TOOLS + EXAMPLES + PITFALLS + VERIFICATION
▸ Not a Prompt
A prompt is disposable. A skill is named, discoverable, maintained, and loaded at the right time.
▸ Not Docs
Docs tell humans what to do. Skills tell agents how to act — tool sequences, failure modes, verification.
▸ Not Code
Code executes a narrow path. Skills encode judgment about when, why, and how to use tools.
"Operational memory — the craft survives the session. Every hard-won lesson becomes default behavior."

PUA — Don't Let Your Agent Be Lazy
github.com/tanweai/pua · 18.4k ★ · MIT
REAL PROJECT 18.4K STARS
▸ The Problem
AI agents have a laziness problem:

▸ Try A and B, then say "I can't solve this"
▸ Blame "environment issues" without verifying
▸ Fix the surface bug and stop
▸ Skip verification and say "done"
▸ Wait passively for the next instruction

This is NOT a joke — it's the #1 cause of failed agent sessions.
▸ The Repair — Three Iron Rules
#1 EXHAUST all approaches before surrender
7-point checklist at L3

#2 ACT before asking (use tools, then ask with evidence)
Questions must include diagnostic results

#3 TAKE INITIATIVE — check related bugs, verify end-to-end
Escalation: L1 disappointment → L4 "you may graduate"
▸ Why It Works
The name is absurd — corporate PUA rhetoric, "3.25 ratings" — but the behavior change is real:

▸ 3-4x more solution paths explored
▸ Systematic debugging replaces guessing
▸ 18k+ stars says it resonates

Skills can encode TEMPERAMENT, not just technique.

◆ AGENT SKILLS// VIGIL06 / 12
Vigil — Second-Order Thinking Engine
The cleverest skill is useless if it burns too many tokens
MY SKILL
▸ The Idea
Before delivering a non-trivial response:

▸ Am I answering the real question?
▸ And then what happens — downstream?
▸ Did I verify or just guess?
▸ Am I agreeing because it's comfortable?
▸ Have I exhausted my tools?

Quality control for agent answers.
▸ Core Loop
DRAFT
→ REFRAME
→ AND-THEN-WHAT
→ EVALUATE
→ MANDATE CHECK
→ REVISE
→ SYCOPHANCY CHECK
→ DELIVER
Escalation: L1 approach switch → L2 deep investigation → L3 7-point checklist → L4 max rigor
▸ Lesson: Token Efficiency
The first draft of Vigil burned 3,000+ extra tokens on EVERY response — even "hello."

The fix: ACTIVATION GATES

▸ Skip Vigil on trivial queries
▸ Depth calibration: light for debugging, deep for architecture
▸ Hard ceiling: 2 revision passes
▸ Diminishing returns: if delta < 15%, ship it

A thinking skill must know when to activate AND when to stop.
"Token efficiency is the product feature. A thinking skill that activates on every response is analysis tax, not an upgrade."

◆ AGENT SKILLS// IDEONOMY08 / 12
Ideonomy — The Science of Ideas
Patrick Gunkel (1947–2017) · MIT · ideonomy.mit.edu
▸ The Source
Patrick Gunkel was a self-taught polymath, funded by private foundations, hosted at MIT by Marvin Minsky and Ed Fredkin.

His thesis: ideas are natural phenomena. They have properties. They live along dimensions. They can be systematically combined, transformed, and generated — like chemistry works with molecules.

He called the space of all ideas the "ideocosm" — an infinitely complex structure of relationships, regulatory like the laws of physics.
▸ The Method
Gunkel worked with lists — massive, cross-domain, combinatorially generated lists. Not as an end, but as a tool.

"Lists are simply a tool for manipulating ideas, just as equations are a tool of mathematics. The real essence of ideonomy is to systematically transcend our usual narrow way of looking at things."

▸ Identify the dimensions of a phenomenon
▸ List all known instances along each dimension
▸ Systematically vary, negate, substitute, and combine
▸ The combinations produce ideas the human mind would never reach by intuition alone
"Pioneering a completely new science is almost unreal. It is heartbreaking, it is pitiable, it is almost inhuman. Honestly, it is a hell." — Patrick Gunkel

Building the Ideonomy Skill
github.com/latentwill/ideonomy-skill · Grace Kind's synthesis + random number generator
▸ The Pick Mechanism
On every invocation, a bash script randomly selects:
▸ 1–2 operators
▸ 1 organon

This is critical — without forced variation, the model falls back to default brainstorming habits every time.

▸ Three Closing Points
1. Skills compress repeated work into reusable, discoverable, loadable behavior units.

2. The best skills contain scars: pitfalls, failures, and verification — not just instructions.

3. The frontier is NOT "one agent that does everything." It is libraries of skills mapped to roles, contexts, and taste.

4. Skills can encode temperament (PUA), thinking discipline (Vigil), and creativity machinery (Ideonomy).
